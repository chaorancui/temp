# 代理配置指南

本文档汇总了常见开发工具在内网环境下的代理配置方法。包括HTTP/HTTPS代理和SOCKS5代理的配置方式，分别介绍了临时指定和永久配置两种方法。

## 代理URL格式说明

```bash
# HTTP/HTTPS 代理格式
http://[username:password@]proxy_host:proxy_port

# SOCKS5 代理格式
socks5://[username:password@]proxy_host:proxy_port
```

示例:

```bash
http://proxy.company.com:8080
http://user:pass@proxy.company.com:8080
socks5://proxy.company.com:1080
```

## 常见代理环境变量

许多工具支持以下环境变量（不区分大小写）：

| 环境变量                  | 说明                           |
| ------------------------- | ------------------------------ |
| http_proxy / HTTP_PROXY   | HTTP代理地址                   |
| https_proxy / HTTPS_PROXY | HTTPS代理地址                  |
| all_proxy / ALL_PROXY     | 所有协议的代理地址             |
| no_proxy / NO_PROXY       | 不使用代理的地址列表，逗号分隔 |
| ftp_proxy / FTP_PROXY     | FTP代理地址                    |

## 工具配置详解

### wget

wget是文件下载工具，支持HTTP、HTTPS和FTP代理。

1. 临时指定代理

   使用 `-e` 或 `--execute` 参数临时指定代理：

   ```bash
   wget -e use_proxy=yes -e http_proxy=http://proxy.company.com:8080 https://example.com/file.tar.gz
   ```

2. 永久指定代理

   在用户主目录创建 `~/.wgetrc` 配置文件：

   ```bash
   # ~/.wgetrc
   use_proxy = on
   http_proxy = http://proxy.company.com:8080
   https_proxy = http://proxy.company.com:8080
   ftp_proxy = http://proxy.company.com:8080
   no_proxy = localhost, 127.0.0.1, *.local
   ```

3. 注意事项
   - 代理URL需要包含完整的 `http://` 或 `https://` 协议头
   - 多个地址在 `no_proxy` 中用逗号分隔，支持通配符
   - 若代理需要认证：`http_proxy = http://user:password@proxy.company.com:8080`

### curl

curl是强大的数据传输工具，支持多种代理协议。

1. 临时指定代理

   使用 `-x` 参数临时指定代理（推荐 `-x + -U` 组合用于需认证的代理）：

   ```bash
   curl -x http://proxy.company.com:8080 https://example.com

   # 带认证
   curl -x http://proxy.company.com:8080 -U user:password https://example.com
   ```

2. 永久指定代理

   在用户主目录创建 `~/.curlrc` 配置文件：

   ```bash
   # ~/.curlrc
   proxy = http://proxy.company.com:8080
   # 或使用 -w 参数用于特定域名
   proxy-user = user:password
   noproxy = "localhost, 127.0.0.1"
   ```

3. 注意事项
   - 使用 `-x / --proxy` 指定代理，支持HTTP、HTTPS、SOCKS4/5
   - 使用 `-U / --proxy-user` 指定代理认证信息
   - 使用 `--noproxy` 指定不走代理的地址
   - 查看当前配置：`curl --version` 可查看支持的协议

### npm

npm是Node.js包管理器，支持HTTP/HTTPS代理配置。

1. 临时指定代理

   使用 `npm config set` 命令临时指定代理：

   ```bash
   npm config set proxy http://proxy.company.com:8080
   npm config set https-proxy http://proxy.company.com:8080
   npm config set registry http://registry.npm.taobao.org  # 如果使用镜像源
   ```

2. 永久指定代理

   编辑 `~/.npmrc` 配置文件（或运行 `npm config edit`）：

   ```ini
   # ~/.npmrc
   proxy=http://proxy.company.com:8080
   https-proxy=http://proxy.company.com:8080
   registry=https://registry.npmjs.org/
   # 带认证（需要base64编码）
   _auth=dXNlcm5hbWU6cGFzc3dvcmQ=
   always-auth=true
   ```

3. 注意事项

- 如果使用带认证的代理，密码部分需Base64编码
- 可使用公共npm镜像源避免代理问题（如淘宝镜像、npmmirror等）
- 查看当前配置：`npm config list`
- 重置配置：`npm config delete proxy`

### pip

pip是Python包管理工具，支持索引服务器和代理配置。

1. 临时指定代理

   使用 `-i` 参数指定镜像源（推荐），或使用 `--proxy` 参数指定代理：

   ```bash
   # 方法1：使用镜像源（推荐）
   pip install -i https://pypi.tsinghua.edu.cn/simple requests

   # 方法2：使用代理
   pip install --proxy [user:passwd@]proxy.server:port requests
   ```

2. 永久指定代理

   创建 `~/.pip/pip.conf` （Linux/Mac）或 `%APPDATA%\pip\pip.ini`（Windows）：

   ```ini
   [global]
   index-url = https://pypi.tsinghua.edu.cn/simple
   # 或使用代理
   proxy = http://user:password@proxy.company.com:8080
   # 信任特定域名
   trusted-host = mirrors.aliyun.com
   ```

3. 注意事项
   - 推荐使用国内镜像源而非直接配置代理（速度更快）
   - 常用镜像源：清华大学、阿里云、网易云等
   - 若必须使用代理：`pip install --proxy [user:passwd@]proxy.server:port`
   - 查看配置：`pip config list`

### cargo

cargo是Rust包管理工具。默认使用crates.io，可配置代理或源替换。

1. 临时指定代理

   临时修改环境变量（不推荐），更推荐使用源替换：

   ```bash
   export http_proxy=http://proxy.company.com:8080
   export https_proxy=http://proxy.company.com:8080
   cargo build
   ```

2. 永久指定代理

   编辑 `~/.cargo/config.toml` 配置文件（[source] 部分用于源替换）：

   ```toml
   [source.crates-io]
   replace-with = "ustc"

   [source.ustc]
   registry = "sparse+https://mirrors.ustc.edu.cn/crates.io-index/"

   # 或配置代理（如果源替换不适用）
   [http]
   proxy = "http://proxy.company.com:8080"
   ```

3. 注意事项
   - 强烈推荐使用源替换而非代理（性能更好）
   - 常用Rust镜像源：USTC、清华大学、阿里云等
   - `sparse+`协议用于稀疏索引，加快下载
   - 若需认证：`proxy = "http://user:password@proxy.company.com:8080"`

### git

git是版本控制系统，支持HTTP和SSH代理配置。

1. 临时指定代理

   使用 `-c` 参数临时指定代理或修改当前仓库配置：

   ```bash
   # 临时指定（单次有效）
   git -c http.proxy=http://proxy.company.com:8080 clone https://github.com/user/repo.git

   # 修改当前仓库配置
   git config http.proxy http://proxy.company.com:8080
   ```

2. 永久指定代理

   编辑 `~/.gitconfig` 全局配置文件或 `.git/config` 仓库配置：

   ```ini
   # ~/.gitconfig
   [http]
       proxy = http://proxy.company.com:8080
       timeout = 30
   [https]
       proxy = http://proxy.company.com:8080
   [http "https://github.com"]
       proxy = socks5://proxy.company.com:1080
   ```

3. 注意事项
   - 支持为特定域名配置不同的代理
   - SSH仓库可通过ProxyCommand在 `~/.ssh/config` 中配置
   - 可配置socks5代理用于特定主机
   - 查看配置：`git config --global --list`

### docker

docker用于容器管理，构建镜像时需要配置代理。

1. 临时指定代理

   在构建时通过 `--build-arg` 传递代理参数：

   ```bash
   docker build \
     --build-arg HTTP_PROXY=http://proxy.company.com:8080 \
     --build-arg HTTPS_PROXY=http://proxy.company.com:8080 \
     -t myimage:latest .
   ```

2. 永久指定代理

   配置Docker守护进程 `/etc/docker/daemon.json`：

   ```json
   {
     "proxies": {
       "default": {
         "httpProxy": "http://proxy.company.com:8080",
         "httpsProxy": "http://proxy.company.com:8080",
         "noProxy": "localhost,127.0.0.1,*.local"
       }
     }
   }
   ```

3. 注意事项
   - 容器内部的代理需要在Dockerfile中配置环境变量
   - 推荐使用国内镜像源而非代理（如中科大、网易等）
   - 重启Docker生效：`systemctl restart docker`
   - 可在 `~/.docker/config.json` 中配置客户端代理

### Maven

Maven是Java项目构建工具，通过settings.xml配置代理。

1. 临时指定代理

   临时指定需要修改 `settings.xml` 或使用系统属性：

   ```bash
   mvn clean install \
     -Dhttp.proxyHost=proxy.company.com \
     -Dhttp.proxyPort=8080 \
     -Dhttps.proxyHost=proxy.company.com \
     -Dhttps.proxyPort=8080
   ```

2. 永久指定代理

   编辑 `~/.m2/settings.xml` 配置文件（<proxies> 部分）：

   ```xml
   <proxies>
     <proxy>
       <id>myproxy</id>
       <active>true</active>
       <protocol>http</protocol>
       <host>proxy.company.com</host>
       <port>8080</port>
       <username>user</username>
       <password>password</password>
       <nonProxyHosts>localhost|127.0.0.1|*.local</nonProxyHosts>
     </proxy>
   </proxies>
   ```

3. 注意事项
   - 密码建议在settings.xml中加密存储
   - 使用settings.xml支持多个代理配置
   - nonProxyHosts使用管道符(|)分隔多个地址
   - 清华大学等提供Maven镜像加速

### Go

Go使用模块管理，通过GOPROXY环境变量配置代理。

1. 临时指定代理

   使用 `export` 临时设置环境变量：

   ```bash
   export GOPROXY=http://proxy.company.com:8080,direct
   export GOSUMDB=off  # 如果需要跳过checksum验证
   go get github.com/example/package
   ```

2. 永久指定代理

   添加到Shell配置文件（`~/.bashrc` 或 `~/.zshrc`）：

   ```bash
   # 在 ~/.bashrc 或 ~/.zshrc 末尾添加
   export GOPROXY=http://proxy.company.com:8080,direct
   export GOSUMDB=off

   # 或使用国内镜像
   export GOPROXY=https://goproxy.cn,direct
   ```

3. 注意事项
   - 常用公开GOPROXY服务：goproxy.cn、athens.azurewebsites.net等
   - GOPROXY格式：支持多个源用逗号分隔，direct表示直接访问
   - 若跳过checksum验证需设置 `GOSUMDB=off`
   - 设置后需重新加载Shell配置：`source ~/.bashrc`

## 配置建议与最佳实践

- **优先使用镜像源而非代理**：国内镜像源通常更快更稳定，避免代理的开销
- **分离临时和永久配置**：临时调试使用命令行参数，正式使用才写入配置文件
- **妥善保管密码**：若代理需认证，密码应加密存储或使用专用凭据管理工具
- **验证连接**：配置后先测试一个简单的请求，确认代理正常工作
- **监控超时**：内网代理可能较慢，根据实际情况调整超时参数
- **记录配置位置**：为团队成员提供一份统一的配置指南，避免重复配置
- **版本管理**：对于项目级的配置（如 `.npmrc`、`gradle.properties`）应纳入版本控制
- **定期更新镜像源URL**：镜像源地址可能变更，需定期检查和更新

## 故障排除

**常见问题**:

| 问题             | 解决方案                                                         |
| ---------------- | ---------------------------------------------------------------- |
| 连接超时         | ping/telnet 代理地址:端口 确认网络连接，检查防火墙规则           |
| 认证失败         | 确认用户名密码正确，若包含特殊字符需URL编码（@ 变为 %40）        |
| SSL证书错误      | 公司内网代理可能拦截HTTPS，需导入证书或使用 --insecure（仅调试） |
| 间歇性失败       | 增加超时时间或使用代理池，检查代理端是否有速率限制               |
| 某个工具配置无效 | 确认工具是否真的支持该代理方式，查看工具文档或版本               |

**调试技巧**:

- **查看详细日志**：大多数工具支持 `-v` 或 `--verbose` 参数
- **抓包分析**：使用 `tcpdump` 或 `Wireshark` 观察网络流量
- **代理日志**：检查代理服务器日志了解请求是否到达代理
- **环境变量验证**：`echo $http_proxy` 确认环境变量是否正确设置
- **配置文件验证**：使用工具的 `config` 或 `show-config` 命令查看实际配置

## 快速参考

**查看当前配置**:

```bash
# wget
cat ~/.wgetrc

# curl
cat ~/.curlrc

# npm
npm config list

# pip
pip config list

# cargo
cat ~/.cargo/config.toml

# git
git config --global --list

# Go
echo $GOPROXY
```

**重置代理配置**:

```bash
# npm
npm config delete proxy
npm config delete https-proxy

# pip
pip config unset global.proxy

# git
git config --global --unset http.proxy

# Go
unset GOPROXY
unset GOSUMDB
```
